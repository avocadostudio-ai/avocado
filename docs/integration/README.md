# Integration Docs (Start Here)

Use this page as the entry point for connecting the Avocado Studio to your website.

## Who is this for?

- **Site owners** who want to add chat-driven AI editing to an existing website.
- **Frontend developers** integrating the editor into a Next.js (or other framework) project.
- **Platform teams** evaluating the editor for enterprise CMS integration.

## Prerequisites

- A running Next.js site (App Router, Next.js 15+)
- Node.js >= 20 and pnpm >= 9
- The Avocado Studio monorepo cloned and running locally (see [root README](../../README.md))
- An OpenAI or Anthropic API key (optional â€” deterministic demo mode works without one)

## The `@ai-site-editor/site-sdk` package

All Next.js integration logic is provided by `@ai-site-editor/site-sdk`. The SDK exports:

- **Route handler factories** â€” `createDraftEnableHandler()`, `createDraftDisableHandler()`, `createBlocksHandler()`, `createPagesHandler()` â€” one-liner API route setup
- **Editor context** â€” `resolveEditorContext()`, `fetchEditorPage()`, `fetchEditorSlugs()`
- **UI components** â€” `EditorOverlay`, `BlockErrorBoundary`
- **Utilities** â€” `buildSlug()`, `single()`, `getPreviewWrapperProps()`
- **Types** â€” `PageDoc`, `PageMeta`, `BlockInstance`, `DraftContext`

Add the SDK as a workspace dependency:

```bash
pnpm add @ai-site-editor/site-sdk
```

## MVP integration model

- Source of truth: block registry maintained in `@ai-site-editor/shared` (or adopter-owned overrides).
- Transport contract: `GET /api/editor/blocks` manifest JSON generated by the SDK's `createBlocksHandler()`.
- Preview bootstrap: Next.js Draft Mode via SDK handler factories (`createDraftEnableHandler`, `createDraftDisableHandler`).
- Fallback behavior: if manifest is missing/invalid, editor runs in degraded mode (no structural edits).

## Required endpoints

All endpoints are created with one-liner SDK handler factories:

| Endpoint | SDK factory | Purpose |
|---|---|---|
| `GET /api/draft` | `createDraftEnableHandler()` | Enables Next.js draft mode, sets cookies, redirects |
| `GET /api/draft/disable` | `createDraftDisableHandler()` | Disables draft mode, clears cookies, redirects |
| `GET /api/editor/blocks` | `createBlocksHandler()` | Block manifest for structural edits |
| `GET /api/editor/pages` | `createPagesHandler(cb)` | Returns published pages to seed the orchestrator |

## Required environment variables

| Variable | App | Required | Description |
|---|---|---|---|
| `DRAFT_MODE_SECRET` | Site + Editor | Yes | Shared secret for draft mode activation. Site also accepts `VITE_SITE_DRAFT_SECRET` â€” set one in a shared `.env`. |
| `ORCHESTRATOR_URL` | Site | No | Orchestrator API base URL (defaults to `http://localhost:4200`) |
| `PUBLISH_TOKEN` | Site | No | Required if publish protection is enabled; must match orchestrator `PUBLISH_TOKEN` |
| `VITE_SITE_ORIGIN` | Editor | Yes | Site origin for iframe target (e.g. `http://localhost:3000`) |
| `VITE_SITE_DRAFT_SECRET` | Editor | Yes | Same value as `DRAFT_MODE_SECRET` (either name works on the site) |
| `VITE_PUBLISH_TOKEN` | Editor | No | Required if publish protection is enabled; must match orchestrator `PUBLISH_TOKEN` |
| `ACCESS_PASSWORD_HASH` | Orchestrator | No | Enables access gate for protected chat/ops endpoints |
| `PUBLISH_TOKEN` | Orchestrator | No | Validates publish requests; should match Site `PUBLISH_TOKEN` and Editor `VITE_PUBLISH_TOKEN` |

## Adoption checklist

1. Install `@ai-site-editor/site-sdk` as a dependency.
2. Create API route files using SDK handler factories (4 one-liner files).
3. Wire `resolveEditorContext()` and `fetchEditorPage()` into your page data loading.
4. Add `EditorOverlay`, `BlockErrorBoundary`, and `getPreviewWrapperProps()` to your page renderer.
5. Set `DRAFT_MODE_SECRET` env var.
6. Confirm editor header shows `Manifest` (not `Degraded`).
