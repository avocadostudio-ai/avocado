## What is AI Site Editor?

AI Site Editor is an open-source, AI-powered content operations platform for websites. Describe content changes in natural language or use the visual drag-and-drop editor — the system plans and applies them as validated, schema-checked operations with live preview, undo/redo, and one-click publishing.

### Two editing modes

<CardGroup cols={2}>
  <Card title="Chat Editor" icon="message-bot">
    The default experience — a split-pane interface where your live site renders on the left and a chat interface on the right. Type requests like _"add a testimonials section below the hero"_ and the AI plans and applies structured edits.
  </Card>
  <Card title="Puck Mode" icon="hand-pointer" href="/integration/puck-mode">
    A visual drag-and-drop editor powered by Puck. Reorder blocks, edit fields directly, and use the AI chat sidebar for natural-language edits — all in one canvas. Enabled per-site via settings.
  </Card>
</CardGroup>

Both modes share the same block system, operation pipeline, and orchestrator backend. The editing mode is a per-site setting — each site can independently opt into Puck mode.

**For developers and agencies**: integrate AI editing into any Next.js site with the Site SDK, or use the framework-agnostic REST API. Self-hosted, multi-model AI (Anthropic, OpenAI, Google Gemini), and a pluggable publishing pipeline.

**For site owners**: manage your website content through conversation — no code changes required. Review every change before it ships.

## Key Capabilities

- **Natural language editing** — Describe changes in plain English. The AI generates structured, schema-validated operations
- **Multi-model AI** — Anthropic, OpenAI, and Google Gemini. Most battle-tested with Anthropic models (Haiku, Sonnet, Opus)
- **20 built-in block types** — Hero, CTA, FAQ, Testimonials, Gallery, Stats, Carousel, Table, and more
- **Type-safe operations** — Zod schemas validate every edit. Malformed edits are rejected before they touch your content
- **Live preview + streaming** — See edits applied progressively as the AI generates the plan
- **Undo/redo + plan approval** — Full operation history. Review and approve every change
- **Pluggable publishing** — Ships with Git + Vercel deploy hooks. Implement the [`PublishTarget` interface](/how-it-works#the-publishtarget-interface) for your own workflow
- **Self-hosted** — Run the entire stack on your own infrastructure. No vendor lock-in
